Problem (Combination Sum II)
Given an array candidates (may contain duplicates) and a target, find all unique combinations that sum to target.
Each number can be used at most once.
Combinations must be unique (no duplicates).
Return all valid combinations.
Order within combinations doesn’t matter.


---

Solution Steps (Backtracking + Sorting):

1. Sort the array to handle duplicates.


2. Use recursive function with index and current sum.


3. If sum == target → add combination to result.


4. If sum > target → backtrack.


5. Iterate from current index to end.


6. Skip duplicates: if i > start && nums[i] == nums[i-1] → continue.


7. Include current element.


8. Recurse with i + 1 (no reuse).


9. Backtrack by removing last element.


10. Continue exploring all possibilities.




---

Pattern:
Backtracking (Subset with Duplicates Handling)


---

Advantage over Brute Force:
Avoids duplicate combinations via sorting.
Prunes invalid paths early.
Efficient subset exploration instead of all permutations.


### 41

Problem (First Missing Positive)
Given an unsorted integer array, find the smallest missing positive integer.
Must run in O(n) time and O(1) extra space.
Array may contain negatives and zeros.
Rearrangement of elements is allowed.
Return the smallest positive integer not present.


---

Solution Steps (Cyclic Sort / Index Mapping):

1. Iterate through array.


2. While nums[i] in range [1, n] and not in correct position:


3. Swap nums[i] with nums[nums[i] - 1].


4. Continue until all valid numbers are placed correctly.


5. Traverse array again.


6. First index i where nums[i] != i+1 → return i+1.


7. If all correct → return n+1.




---

Pattern:
Cyclic Sort / Index Placement


---

Advantage over Brute Force:
Achieves O(n) without extra space.
Avoids sorting O(n log n).
Directly maps values to indices efficiently.


### 42

Problem (Trapping Rain Water)
Given an array height[] representing elevation, compute trapped rainwater.
Water is trapped between taller bars.
Width of each bar is 1.
Return total water trapped.
Array may contain varying heights.


---

Solution Steps (Two Pointer):

1. Initialize left = 0, right = n-1.


2. Track leftMax = 0, rightMax = 0.


3. While left < right:


4. If height[left] < height[right]:


5. If height[left] >= leftMax → update leftMax.


6. Else add leftMax - height[left] to result.


7. Move left++.


8. Else process right side similarly.


9. Move right--.


10. Return total water.




---

Pattern:
Two Pointers (Prefix Max Optimization)


---

Advantage over Brute Force:
Avoids nested scanning O(n²).
Achieves O(n) with constant space.
Efficiently computes water using boundaries.
