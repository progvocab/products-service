Problem (Pow(x, n))
Implement pow(x, n) to compute x raised to the power n.
n can be negative.
Return result as a double.
Handle large values efficiently.
Avoid using built-in power functions.


---

Solution Steps (Binary Exponentiation):

1. If n == 0 → return 1.


2. Convert n to long to handle overflow.


3. If n < 0 → set x = 1/x, n = -n.


4. Initialize result = 1.


5. While n > 0:


6. If n is odd → result *= x.


7. Square x = x * x.


8. Halve n = n / 2.


9. Continue until n == 0.


10. Return result.




---

Pattern:
Divide & Conquer (Binary Exponentiation)


---

Advantage over Brute Force:
Reduces from O(n) to O(log n).
Efficient for very large powers.
Minimizes multiplications drastically.

Problem (N-Queens)
Place n queens on an n x n chessboard such that no two attack each other.
Queens attack in rows, columns, and diagonals.
Return all distinct valid board configurations.
Each solution is represented as a list of strings.
Order of solutions does not matter.


---

Solution Steps (Backtracking):

1. Use recursion row by row.


2. Track columns, diagonals, anti-diagonals using sets/arrays.


3. For each row, try placing queen in each column.


4. If column/diagonal is safe → place queen.


5. Mark column and diagonals as occupied.


6. Recurse to next row.


7. If all rows filled → add board to result.


8. Backtrack: remove queen and unmark.


9. Continue exploring all possibilities.




---

Pattern:
Backtracking (Constraint Satisfaction)


---

Advantage over Brute Force:
Prunes invalid placements early.
Avoids checking all nⁿ configurations.
Efficient constraint tracking reduces search space.

### 53

Problem (Maximum Subarray)
Given an integer array, find the contiguous subarray with the largest sum.
Return the maximum sum.
Array may contain negative numbers.
At least one element exists.
Subarray must be non-empty.


---

Solution Steps (Kadane’s Algorithm):

1. Initialize currentSum = nums[0], maxSum = nums[0].


2. Iterate from index 1 to n-1.


3. Update currentSum = max(nums[i], currentSum + nums[i]).


4. Update maxSum = max(maxSum, currentSum).


5. Continue till end of array.


6. Return maxSum.




---

Pattern:
Dynamic Programming (Kadane’s Algorithm)


---

Advantage over Brute Force:
Reduces from O(n²) to O(n).
No need to check all subarrays.
Efficient running sum optimization.


### 54

Problem (Spiral Matrix)
Given an m x n matrix, return all elements in spiral order.
Traverse from top-left in clockwise direction.
Process layer by layer.
Return the traversal as a list.
Matrix may not be square.


---

Solution Steps (Boundary Traversal):

1. Initialize top = 0, bottom = m-1, left = 0, right = n-1.


2. Traverse left → right across top row.


3. Increment top.


4. Traverse top → bottom along right column.


5. Decrement right.


6. If top <= bottom, traverse right → left across bottom.


7. Decrement bottom.


8. If left <= right, traverse bottom → top along left column.


9. Increment left.


10. Repeat until boundaries cross.




---

Pattern:
Matrix Simulation (Spiral Traversal)


---

Advantage over Brute Force:
Avoids revisiting elements.
Single pass → O(m * n).
Efficient boundary control.


### 55

Problem (Jump Game)
Given an array nums, each element represents max jump length.
Start at index 0, determine if you can reach last index.
Return true if reachable, else false.
Array may contain zeros.
Optimize for linear time.


---

Solution Steps (Greedy):

1. Initialize farthest = 0.


2. Traverse array from index 0.


3. If i > farthest → return false.


4. Update farthest = max(farthest, i + nums[i]).


5. If farthest >= last index → return true.


6. Continue traversal.


7. Return true if end is reachable.




---

Pattern:
Greedy (Reachability)


---

Advantage over Brute Force:
Avoids exploring all paths (exponential).
Linear time O(n).
Tracks maximum reachable index efficiently.

Pattern:
Sorting + Greedy Merge

Single pass merging after sort.

Given an integer n, generate an n x n matrix filled with numbers from 1 to n² in spiral order.
Start from top-left and move clockwise.
Fill all cells exactly once.
Return the generated matrix.
Matrix is always square.


---

Solution Steps (Boundary Traversal):

1. Initialize matrix and boundaries: top, bottom, left, right.


2. Set num = 1.


3. Traverse left → right across top row.


4. Increment top.


5. Traverse top → bottom along right column.


6. Decrement right.


7. Traverse right → left across bottom row.


8. Decrement bottom.


9. Traverse bottom → top along left column.


10. Increment left, repeat until filled.




---

Pattern:
Matrix Simulation (Spiral Traversal)


---

Advantage over Brute Force:
Systematic fill avoids backtracking.
Single pass → O(n²).
Efficient boundary control prevents revisits.
