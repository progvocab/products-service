### 60

Problem (Permutation Sequence)
Given n and k, return the kth permutation of numbers 1 to n.
Permutations are ordered lexicographically.
n ≤ 9 (factorial manageable).
Return result as a string.
Avoid generating all permutations.


---

Solution Steps (Factorial Number System):

1. Precompute factorials up to n.


2. Create list of numbers [1..n].


3. Convert k = k - 1 (0-based index).


4. For each position:


5. Find index = k / factorial[n-1].


6. Append number at index to result.


7. Remove that number from list.


8. Update k = k % factorial[n-1].


9. Decrease n and repeat.


10. Return result string.




---

Pattern:
Math (Factorial / Permutation Indexing)


---

Advantage over Brute Force:
Avoids generating all n! permutations.
Runs in O(n²) due to list removals.
Direct computation using factorial representation.



### 61

Problem (Rotate List)
Given a linked list, rotate it to the right by k places.
Each rotation moves last node to front.
Return the new head.
List may be empty.
k can be greater than list length.


---

Solution Steps (Cycle + Break):

1. If head is null or single node → return head.


2. Traverse list to find length n and last node.


3. Connect last node to head → make it circular.


4. Compute k = k % n.


5. Find new tail at position n - k - 1.


6. New head is newTail.next.


7. Break the circle: newTail.next = null.


8. Return new head.




---

Pattern:
Linked List (Cycle Formation)


---

Advantage over Brute Force:
Avoids rotating k times O(n*k).
Single traversal → O(n).
Efficient use of circular structure.
