Problem (Substring with Concatenation of All Words)
Given a string s and an array of words (same length), find all starting indices of substrings formed by concatenating all words exactly once.
Words can appear in any order.
Each word must be used exactly once.
No extra characters allowed in between.
Return all valid starting indices.


---

Solution Steps (Sliding Window + HashMap):

1. Store word frequency in a hashmap.


2. Let wordLen, numWords, windowSize = wordLen * numWords.


3. Iterate over wordLen starting offsets.


4. Use sliding window with left pointer.


5. Extract words of size wordLen.


6. Maintain current window frequency map.


7. If word count exceeds → shrink window.


8. If window matches required frequency → record index.


9. Continue sliding for all offsets.




---

Pattern:
Sliding Window + Hashing


---

Advantage over Brute Force:
Avoids checking all permutations.
Efficient O(n * wordLen).
Uses hashmap for quick frequency validation.

Problem (Next Permutation)
Given an array of integers, rearrange it into the next lexicographically greater permutation.
If no such permutation exists, rearrange it to the lowest possible order (sorted ascending).
Modification must be in-place.
Use only constant extra memory.
Return the modified array.


---

Solution Steps (Greedy + Reverse):

1. Find first index i from right where nums[i] < nums[i+1].


2. If no such index → reverse entire array.


3. Find index j from right where nums[j] > nums[i].


4. Swap nums[i] and nums[j].


5. Reverse subarray from i+1 to end.


6. This gives next lexicographic permutation.


7. Done in-place.




---

Pattern:
Greedy + Array Manipulation


---

Advantage over Brute Force:
Avoids generating all permutations.
Runs in O(n) time.
Uses constant extra space.

### 32

Problem (Longest Valid Parentheses)
Given a string containing only ( and ), find the length of the longest valid (well-formed) parentheses substring.
Substring must be contiguous.
Return the maximum length.
String may contain invalid sequences.
Handle nested and overlapping cases.


---

Solution Steps (Stack):

1. Initialize stack with -1 (base index).


2. Traverse string with index i.


3. If ( → push index i.


4. If ) → pop from stack.


5. If stack becomes empty → push current index i.


6. Else calculate length = i - stack.peek().


7. Update max length.


8. Continue till end.


9. Return max length.




---

Pattern:
Stack (Index Tracking)


---

Advantage over Brute Force:
Avoids checking all substrings O(n³).
Efficient single pass → O(n).
Handles nested structures naturally.


### 33

Problem (Search in Rotated Sorted Array)
Given a rotated sorted array (no duplicates) and a target, return its index.
If not found, return -1.
Array was originally sorted in ascending order.
Rotation is at an unknown pivot.
Must run in O(log n) time.


---

Solution Steps (Modified Binary Search):

1. Initialize left = 0, right = n-1.


2. While left <= right:


3. Compute mid.


4. If nums[mid] == target → return mid.


5. Check which half is sorted (left or right).


6. If left half sorted and target lies in it → move right = mid - 1.


7. Else move left = mid + 1.


8. If right half sorted and target lies in it → move left = mid + 1.


9. Else move right = mid - 1.


10. Return -1 if not found.




---

Pattern:
Binary Search (Modified)


---

Advantage over Brute Force:
Reduces from O(n) to O(log n).
Uses sorted property even after rotation.
Efficient pivot handling implicitly.


### 34

Problem (Find First and Last Position of Element in Sorted Array)
Given a sorted array and a target, find the starting and ending position of the target.
Return [-1, -1] if target is not found.
Array is sorted in non-decreasing order.
Must run in O(log n) time.
Return indices of first and last occurrence.


---

Solution Steps (Binary Search Twice):

1. Perform binary search to find first occurrence.


2. Move left boundary when match found.


3. Store first index.


4. Perform binary search to find last occurrence.


5. Move right boundary when match found.


6. Store last index.


7. If not found → return [-1, -1].


8. Return [first, last].




---

Pattern:
Binary Search (Lower Bound & Upper Bound)


---

Advantage over Brute Force:
Avoids linear scan O(n).
Efficient O(log n) solution.
Precisely locates boundaries using binary search.

### 35

Problem (Search Insert Position)
Given a sorted array and a target, return its index if found.
If not found, return the index where it should be inserted.
Array contains distinct integers.
Must run in O(log n) time.
Maintain sorted order after insertion.


---

Solution Steps (Binary Search):

1. Initialize left = 0, right = n-1.


2. While left <= right:


3. Compute mid.


4. If nums[mid] == target → return mid.


5. If nums[mid] < target → left = mid + 1.


6. Else → right = mid - 1.


7. Loop ends when left > right.


8. Return left as insertion index.




---

Pattern:
Binary Search


---

Advantage over Brute Force:
O(log n) vs O(n) linear scan.
Efficient for large arrays.
Directly finds correct insertion point.


### 36

Problem (Valid Sudoku)
Determine if a 9×9 Sudoku board is valid.
Check rows, columns, and 3×3 sub-boxes.
Digits 1–9 must not repeat in any unit.
Empty cells are represented by ..
Only validate current board (not solve it).


---

Solution Steps (Hashing / Sets):

1. Use 3 structures: rows[9], cols[9], boxes[9] as sets.


2. Traverse each cell (i, j).


3. If cell is . → continue.


4. Compute box index: (i/3)*3 + (j/3).


5. If value exists in row/col/box → return false.


6. Else add value to respective sets.


7. Continue till all cells processed.


8. Return true.




---

Pattern:
Hashing (Set Validation)


---

Advantage over Brute Force:
Avoids repeated scans per row/col/box.
Single pass → O(1) (fixed 9×9 grid).
Efficient constraint checking.

Each next term describes the previous term’s digits.
“Count and say” the number of digits in groups.

---

Solution Steps (String Construction):

2. Repeat from 2 to n:


3. Initialize empty temp string.


4. Traverse current result string.


5. Count consecutive same digits.


6. Append count + digit to temp.


7. Move to next different digit.


8. Update result = temp.


9. After n iterations, return result.




---

Pattern:
String Simulation / Iterative Construction


---

Advantage over Brute Force:

Linear processing per term.
Avoids recomputation of previous terms.

### 39

Problem (Combination Sum)
Given an array of distinct integers candidates and a target, find all unique combinations that sum to target.
You can use the same number unlimited times.
Combinations must be unique.
Return all valid combinations.
Order within combinations doesn’t matter.


---

Solution Steps (Backtracking):

1. Sort candidates (optional for pruning).


2. Use recursive function with current sum and index.


3. If sum == target → add combination to result.


4. If sum > target → backtrack.


5. Iterate from current index to end.


6. Include candidate[i] in current combination.


7. Recurse with same index (reuse allowed).


8. Backtrack by removing last element.


9. Continue exploring all paths.




---

Pattern:
Backtracking (Unbounded Choice)


---

Advantage over Brute Force:
Prunes paths when sum exceeds target.
Avoids generating all subsets blindly.
Efficient exploration of valid combinations only.
