### 70

Problem (Climbing Stairs)
You are climbing n stairs.
At each step, you can climb either 1 or 2 stairs.
Find the total number of distinct ways to reach the top.
n is a positive integer.
Return the count of ways.


---

Solution Steps (Dynamic Programming):

1. If n <= 2, return n.


2. Initialize prev2 = 1, prev1 = 2.


3. Iterate from 3 to n.


4. Compute curr = prev1 + prev2.


5. Update prev2 = prev1.


6. Update prev1 = curr.


7. Continue till n.


8. Return prev1.




---

Pattern:
Dynamic Programming (Fibonacci Pattern)


---

Advantage over Brute Force:
Avoids exponential recursion.
Optimized to O(n) time and O(1) space.
Reuses previously computed results.

### 71

Problem (Simplify Path)
Given a Unix-style absolute file path, simplify it.
Handle ".", "..", and multiple slashes correctly.
"." means current directory.
".." means move to parent directory.
Return the canonical simplified path.


---

Solution Steps (Stack):

1. Split path using /.


2. Initialize an empty stack.


3. Traverse each directory token.


4. Ignore empty strings and ".".


5. If token is ".." → pop stack if not empty.


6. Else push valid directory name into stack.


7. Join stack elements with /.


8. Prefix result with /.


9. Return simplified path.




---

Pattern:
Stack (Path Processing)


---

Advantage over Brute Force:
Single traversal → O(n).
Efficient directory backtracking using stack.
Avoids complex string manipulations repeatedly.


### 72

Problem (Edit Distance)
Given two strings word1 and word2, find the minimum operations to convert word1 into word2.
Allowed operations: insert, delete, replace.
Each operation costs 1.
Return the minimum edit distance.
Strings may have different lengths.


---

Solution Steps (Dynamic Programming):

1. Create DP table of size (m+1) x (n+1).


2. Initialize first row/column with index values.


3. Traverse both strings.


4. If characters match → dp[i][j] = dp[i-1][j-1].


5. Else take minimum of insert, delete, replace.


6. Formula: 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]).


7. Fill table row by row.


8. Return dp[m][n].




---

Pattern:
Dynamic Programming (String Transformation)


---

Advantage over Brute Force:
Avoids exponential recursive comparisons.
Efficient O(m * n) computation.
Stores overlapping subproblem results.

### 73

Problem (Set Matrix Zeroes)
Given an m x n matrix, if an element is 0, set its entire row and column to 0.
Modify the matrix in-place.
Avoid using extra matrix space.
All updates should happen simultaneously.
Return the modified matrix.


---

Solution Steps (In-Place Marking):

1. Use first row and first column as markers.


2. Track whether first row/column originally contain zero.


3. Traverse matrix (excluding first row/column).


4. If matrix[i][j] == 0:


5. Mark matrix[i][0] = 0 and matrix[0][j] = 0.


6. Traverse again to set cells to 0 using markers.


7. Process first row if needed.


8. Process first column if needed.


9. Return modified matrix.




---

Pattern:
Matrix Manipulation (In-Place Marking)


---

Advantage over Brute Force:
Avoids extra O(m*n) storage.
Efficient O(1) extra space.
Single marking strategy minimizes repeated scans.
