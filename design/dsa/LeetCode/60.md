





### 60

Problem (Permutation Sequence)
Given n and k, return the kth permutation of numbers 1 to n.
Permutations are ordered lexicographically.
n ≤ 9 (factorial manageable).
Return result as a string.
Avoid generating all permutations.


---

Solution Steps (Factorial Number System):

1. Precompute factorials up to n.


2. Create list of numbers [1..n].


3. Convert k = k - 1 (0-based index).


4. For each position:


5. Find index = k / factorial[n-1].


6. Append number at index to result.


7. Remove that number from list.


8. Update k = k % factorial[n-1].


9. Decrease n and repeat.


10. Return result string.




---

Pattern:
Math (Factorial / Permutation Indexing)


---

Advantage over Brute Force:
Avoids generating all n! permutations.
Runs in O(n²) due to list removals.
Direct computation using factorial representation.



### 61

Problem (Rotate List)
Given a linked list, rotate it to the right by k places.
Each rotation moves last node to front.
Return the new head.
List may be empty.
k can be greater than list length.


---

Solution Steps (Cycle + Break):

1. If head is null or single node → return head.


2. Traverse list to find length n and last node.


3. Connect last node to head → make it circular.


4. Compute k = k % n.


5. Find new tail at position n - k - 1.


6. New head is newTail.next.


7. Break the circle: newTail.next = null.


8. Return new head.




---

Pattern:
Linked List (Cycle Formation)


---

Advantage over Brute Force:
Avoids rotating k times O(n*k).
Single traversal → O(n).
Efficient use of circular structure.



### 62

Problem (Unique Paths)
Given m x n grid, count unique paths from top-left to bottom-right.
You can only move right or down.
Return total number of distinct paths.
Grid has no obstacles.
Result fits within integer range.


---

Solution Steps (Dynamic Programming):

1. Create DP array of size m x n.


2. Initialize first row and column as 1.


3. For each cell (i,j):


4. dp[i][j] = dp[i-1][j] + dp[i][j-1].


5. Fill the grid row by row.


6. Final answer is dp[m-1][n-1].


7. Can optimize to 1D array if needed.




---

Pattern:
Dynamic Programming (Grid Paths)


---

Advantage over Brute Force:
Avoids exponential recursion.
Efficient O(m * n) solution.
Reuses subproblem results.


### 63


Problem (Unique Paths II)
Given an m x n grid with obstacles, count unique paths from top-left to bottom-right.
Cells with 1 are obstacles, 0 are free.
You can only move right or down.
Return number of unique paths avoiding obstacles.
Start or end may be blocked.


---

Solution Steps (Dynamic Programming):

1. Create DP array same size as grid.


2. If start is obstacle → return 0.


3. Initialize dp[0][0] = 1.


4. Fill first row/column considering obstacles.


5. For each cell (i,j):


6. If obstacle → dp[i][j] = 0.


7. Else dp[i][j] = dp[i-1][j] + dp[i][j-1].


8. Continue filling grid.


9. Return dp[m-1][n-1].




---

Pattern:
Dynamic Programming (Grid with Constraints)


---

Advantage over Brute Force:
Avoids exponential path exploration.
Efficient O(m * n) computation.
Handles obstacles via state transitions.


### 64


Problem (Minimum Path Sum)
Given an m x n grid filled with non-negative numbers, find a path from top-left to bottom-right with minimum sum.
You can only move right or down.
Return the minimum path sum.
All values are non-negative.
Path must be continuous.


---

Solution Steps (Dynamic Programming):

1. Use the grid itself or a DP array.


2. Initialize first row and column with cumulative sums.


3. For each cell (i,j):


4. grid[i][j] += min(grid[i-1][j], grid[i][j-1]).


5. Fill the grid row by row.


6. Final answer is at grid[m-1][n-1].


7. Return that value.




---

Pattern:
Dynamic Programming (Grid Optimization)


---

Advantage over Brute Force:
Avoids exploring all paths (exponential).
Efficient O(m * n) solution.
Reuses optimal substructure.





### 65

Problem (Valid Number)
Given a string, determine if it represents a valid number.
It may include integers, decimals, and exponent (e/E).
Spaces may appear at start/end.
Signs (+/-) allowed in specific positions.
Return true if valid, else false.


---

Solution Steps (State Tracking):

1. Trim leading/trailing spaces.


2. Initialize flags: numSeen, dotSeen, eSeen, numAfterE.


3. Traverse each character.


4. If digit → set numSeen = true, numAfterE = true.


5. If . → invalid if dotSeen or eSeen.


6. If e/E → invalid if eSeen or no number before.


7. Set eSeen = true, numAfterE = false.


8. If +/- → valid only at start or after e/E.


9. Else → invalid character → return false.


10. Return numSeen && numAfterE.




---

Pattern:
String Parsing (Finite State / Flags)


---

Advantage over Brute Force:
Single pass → O(n).
Avoids complex parsing or regex overhead.
Efficient state validation with minimal space.
